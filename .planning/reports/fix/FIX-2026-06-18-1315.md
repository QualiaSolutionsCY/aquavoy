**Symptom:** No response in the Finance page ("maybe choose a model").

## Feedback Loop
- Initial: inspection — Finance posts to the same `/api/chat` as the working chat, so the differentiator is the request shape, not the model. The Finance "scan & propose" prompt drives a long multi-tool OneDrive loop.

## Root Cause
`src/app/api/chat/route.ts:6-7` — the route declared `runtime`/`dynamic` but no `maxDuration`, so it ran under Vercel's short default function timeout. `streamChatWithTools` runs up to `MAX_TOOL_ITERATIONS = 10` non-streaming tool calls (OneDrive list/search/read) BEFORE the first streamed byte. A normal chat turn (0–1 tool calls) finishes inside the default; the Finance scan does not, so the function is killed before the streamed reply starts and the page renders no response. The model is irrelevant — chat and finance share it; only the tool-loop duration differs.

## Patch
- Files changed: `src/app/api/chat/route.ts` (1 file)
- Change summary: added `export const maxDuration = 300;` (with rationale comment) to match the 120s stream-header window in `lib/openrouter/client.ts` with headroom. Requires the Vercel Pro plan (Qualia's teams are Pro).
